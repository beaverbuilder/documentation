Use Menu to render a WordPress navigation menu with horizontal, vertical, accordion, or expanded layouts and a configurable responsive toggle.

## Usage

Use Menu to render a WordPress menu created under **Appearance > Menus** as a fully styled navigation. The module supports horizontal, vertical, accordion, and expanded layouts, optional submenu indicators, an inline logo for centered horizontal menus, an optional search field, and an optional WooCommerce cart item. It also handles responsive behavior with a hamburger, labeled hamburger, text button, or always-expanded toggle, and supports inline, below-row, overlay, and flyout reveal styles.

Use Menu in headers, footers, sidebars, and saved templates that need a configurable site navigation without a separate menu plugin. It fits theme builder header layouts, mobile-first navigation patterns, and pages that combine navigation with search or cart entry points.

## Module Settings

The Menu module settings control which WordPress menu is rendered, its layout and submenu behavior, optional centered logo, search and WooCommerce additions, the responsive toggle, and the appearance of links, dropdowns, and the responsive UI.

### General Tab

The General tab selects the source menu, the desktop layout, submenu icons, and the responsive toggle behavior, and configures the optional inline logo and search additions.

<details>
<summary><strong>Menu</strong></summary>


Selects a WordPress menu created in the admin under **Appearance > Menus**. When no menus exist, the field shows a link to add one.
</details>

<details>
<summary><strong>Layout</strong> <small>Default: Horizontal</small></summary>


Sets the desktop layout used to render the menu.

- **Horizontal:** Lays the menu out in a single row with hover-revealed dropdowns.
- **Vertical:** Stacks menu items in a column with hover-revealed dropdowns.
- **Accordion:** Stacks menu items in a column with click-revealed dropdowns suitable for sidebars and mobile patterns.
- **Expanded:** Renders all submenu items inline beneath their parent without a hover or click toggle.

<details>
<summary><strong>Horizontal Settings</strong></summary>


<details>
<summary><strong>Submenu Icon</strong> <small>Default: None</small></summary>


Selects the indicator displayed next to items that have a submenu when **Layout** is set to **Horizontal** or **Vertical**.

- **Arrows:** Shows a directional arrow next to items with a submenu.
- **Plus sign:** Shows a plus sign next to items with a submenu.
- **None:** Hides the submenu indicator.

</details>
</details>

<details>
<summary><strong>Accordion Settings</strong></summary>


<details>
<summary><strong>Submenu Icon click</strong> <small>Default: Arrows</small></summary>


Selects the indicator displayed next to items that have a submenu when **Layout** is set to **Accordion**.

- **Arrows:** Shows a directional arrow that rotates when the submenu is expanded.
- **Plus sign:** Shows a plus sign that toggles when the submenu is expanded.

</details>

<details>
<summary><strong>Collapse Inactive</strong> <small>Default: Yes</small></summary>


Controls whether multiple accordion items can stay open at the same time. **Yes** keeps only one item open at a time. **No** allows multiple items to be open at the same time.
</details>
</details>
</details>

<details>
<summary><strong>Menu Name</strong> <small>Default: Menu</small></summary>


Used as the menu `aria` attribute for accessibility and as the label shown by the responsive menu toggle.
</details>

<details>
<summary><strong>Centered + Inline Logo</strong></summary>


<details>

The image rendered inline within the top-level menu items.
</details>

<details>

Places the inline logo on the left or right side of odd menu items.
</details>
</details>
</details>

<details>
<summary><strong>Responsive</strong></summary>


The Responsive section configures the toggle used to reveal the menu on smaller screens.

<details>
<summary><strong>Responsive</strong></summary>


<details>
<summary><strong>Responsive Toggle</strong> <small>Default: Hamburger Icon</small></summary>


Selects the control rendered when the responsive breakpoint is met.

- **Hamburger Icon:** Renders an icon-only toggle button.
- **Hamburger Icon + Label:** Renders the hamburger icon next to the **Menu Name** label.
- **Menu Button:** Renders a text-only toggle button using the **Menu Name** label.
- **None:** Skips the toggle and keeps the menu visible at all sizes.

<details>
<summary><strong>Responsive Style</strong></summary>


<details>
<summary><strong>Responsive Style</strong> <small>Default: Inline</small></summary>


Selects how the revealed responsive menu is positioned. This field appears when **Responsive Toggle** is not set to **None**.

- **Inline:** Renders the revealed menu inline with the toggle.
- **Below Row:** Renders the revealed menu below the parent row.
- **Overlay:** Renders the revealed menu as a full-width overlay.
- **Flyout Overlay:** Slides the menu in over the page from the side.
- **Flyout Push:** Slides the menu in from the side and pushes the page content.
- **Flyout Push with Opacity:** Slides the menu in from the side, pushes the page content, and fades the page.

<details>
<summary><strong>Flyout Position</strong></summary>


<details>
<summary><strong>Flyout Position</strong> <small>Default: Left</small></summary>


Sets the side that flyout menus enter from. This field appears when **Responsive Style** is set to a flyout option.
</details>
</details>

<details>
<summary><strong>Flyout Width</strong></summary>


<details>
<summary><strong>Flyout Width</strong> <small>Default: 250</small></summary>


The width of the flyout panel in pixels. This field appears when **Responsive Style** is set to a flyout option.
</details>
</details>
</details>
</details>

<details>
<summary><strong>Responsive Breakpoint</strong></summary>


<details>
<summary><strong>Responsive Breakpoint</strong> <small>Default: Small Devices Only</small></summary>


The viewport size at which the responsive toggle takes over from the desktop layout. This field appears when **Responsive Toggle** is not set to **None**.

- **Always:** Uses the responsive toggle at every screen size.
- **Large, Medium & Small Devices Only:** Switches to the toggle below the large breakpoint.
- **Medium & Small Devices Only:** Switches to the toggle below the medium breakpoint.
- **Small Devices Only:** Switches to the toggle below the responsive breakpoint.

</details>
</details>

<details>
<summary><strong>Stacked Layout</strong></summary>


<details>
<summary><strong>Stacked Layout</strong> <small>Default: Yes</small></summary>


Stacks menu items vertically when **Responsive Toggle** is set to **None**.
</details>
</details>
</details>

<details>
<summary><strong>Sub-menu Item Icon</strong> <small>Default: None</small></summary>


Selects the icon used next to submenu items in the responsive menu.

- **None:** Hides the submenu item icon.
- **Arrow:** Shows an arrow next to submenu items.

</details>
</details>
</details>

<summary><strong>Display Type</strong></summary>


<details>
